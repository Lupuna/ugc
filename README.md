Опиши модели данных для реализации данного приложения

Quiz содержит название, автора и дату создания. Автор - это ссылка на стандартную модель пользователя Django

Внутри опроса находятся Question. У каждого вопроса есть текст и числовое поле order, которое задаёт порядок отображения. Комбинация опроса и порядкового номера уникальна. По этой же паре стоит индекс для быстрой выборки конкретного вопроса по номеру

У каждого вопроса есть QuestionVariant. Они тоже упорядочены через поле order с уникальностью в рамках своего вопроса

Session фиксирует факт прохождения опроса конкретным пользователем. Пара опрос-пользователь уникальна. При необходимости в сессию можно добавить поле статуса, чтобы отслеживать, завершил пользователь прохождение опроса или ещё нет

Session хранится в базе данных, а не в кэше. При 15 миллионах пользователей и примерно 5 опросах на каждого получается около 75 миллионов сессий. Держать такой объём в оперативной памяти нецелесообразно

Answer привязан к сессии и конкретному вопросу. Комбинация уникальна
